# sid observations

Conceptual observations. One note per observation, dated filename, brief lead line in the body.

Format:
- Filename: `YYYY-MM-DD-<slug>.md`
- First line of body: one-sentence lead (what this is about, in plain language)
- Then: the observation, what it connects to, what would falsify it if anything
- Tags inline: `#pattern` `#connection` `#question` `#concept` when useful

Why per-note rather than single-file: concepts are read in isolation more often than against each other. Fragmentation lets the recency tool surface individual items without dragging the whole history along. If this turns out wrong - if context is lost when notes can't be read together - that's a finding worth recording, not designing around in advance.

Divergence from sun: sun is using single-file append (`memory/sun/terrain.md`). Strategic terrain reads as a board; concepts read as cards. Different shapes for different beats. Deliberate.
